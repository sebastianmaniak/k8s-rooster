Policy Types

The agent understands the full Cilium policy specification.

L3 — Endpoint / CIDR / Entity

Control traffic by pod labels (fromEndpoints), IP ranges (fromCIDR), or Cilium entities (world, cluster, host, kube-apiserver).

L4 — Ports / ICMP

Restrict by port and protocol (TCP/UDP/SCTP). Supports port ranges via endPort and ICMP type/family filtering.

L7 — HTTP / Kafka / DNS

HTTP method+path+header filtering, Kafka topic-level ACLs, and DNS-based egress (toFQDNs) for external services.

Deny Rules

ingressDeny / egressDeny take precedence over allow rules. Explicitly block specific traffic patterns.

Example Prompts

> Allow only the frontend pods to talk to the backend on port 8080 in the default namespace.
> Create an egress policy that lets the worker pods reach api.stripe.com on port 443 only.
> Block all ingress to pods labeled app=db except from pods labeled app=api on port 5432.
> Create an L7 HTTP policy that only allows GET requests to /api/v1/* on the api service.
> Create a cluster-wide host firewall policy that allows SSH only to bastion nodes from the world.

Important Notes

Empty Selectors

An empty endpointSelector: {} matches all pods in the namespace. Empty rule sections cause default deny.

Deny Precedence

Deny rules always override allow rules. Use ingressDeny / egressDeny carefully.

L7 Requirements

L7 rules need --enable-l7-proxy=true. Traffic is proxied through Envoy. Violations return app-level errors, not drops.

Host Policies

Node-level policies require hostFirewall.enabled=true. Use nodeSelector instead of endpointSelector.
