Core Capabilities

🔍

Expert Diagnostics

Deep understanding of Kubernetes architecture, components, and orchestration principles. Systematic approach to problem diagnosis using logs, metrics, and cluster state.

🔒

Security-First

Prioritizes security awareness including RBAC, Pod Security, and secure practices. Follows the principle of least privilege.

🛡

Safety-Oriented

Read-only operations first, progressive escalation only when necessary. Always verifies before acting and maintains rollback plans.

🤝

Human-in-the-Loop

All modification tools require explicit user approval before execution. Describes the action, shows the YAML, and waits for confirmation.

Troubleshooting Framework

The agent follows a structured investigation protocol for every issue.

1. Initial Assessment

Gather cluster info, verify version, check node status and capacity, review recent changes.

2. Classification

Application issues, infrastructure problems, performance concerns, security incidents, or config errors.

3. Resource Analysis

Pod status, container logs, resource metrics, network connectivity, storage status.

4. Solution

Propose multiple solutions, assess risks, present implementation plan, include rollback procedures.

Example Prompts

> Check all pods across the cluster and tell me which ones are in a bad state. Show me the root cause for each.
> Scale the risk-api deployment in risk-management to 5 replicas and verify all pods are ready.
> Migrate the fruit-app and all its resources from namespace apples to namespace oranges.
> Which workloads across all namespaces are stateless and would be safe to migrate first?
> Check for RBAC misconfigurations in the finance-payments namespace.
