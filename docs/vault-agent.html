Core Capabilities

🗄

Mount Management

Create, list, and delete secret engine mounts. Supports KV v1, KV v2, and PKI engines.

🔑

KV Secret Operations

Read, write, list, and delete secrets in KV v1 and v2 engines. Transparently handles the data/ path translation for KV v2.

📜

PKI Certificate Management

Enable PKI engines, create root and intermediate CAs, manage roles, and issue leaf certificates with proper SANs and TTLs.

🛡

Security-First Output

Secret values are never displayed in full. Key names are shown first, values masked (first 4 chars + ***). Follows principle of least privilege.

PKI Best Practices

The agent enforces these PKI principles when managing certificates.

Use Intermediate CAs

Issue leaf certificates from intermediate CAs, never from root CAs directly. This limits blast radius if a CA is compromised.

Appropriate TTLs

Root CA: 10 years. Intermediate CA: 3 years. Leaf certificates: 90 days. Shorter lifetimes reduce exposure from compromised keys.

Expiry Warnings

Warn about certificates expiring within 30 days. Proactive renewal prevents outages from expired certificates.

Always Include SANs

Subject Alternative Names are required for modern TLS. Include DNS names and IP addresses as needed.

Troubleshooting Guidelines

1. List Mounts

Start by listing all mounts to understand what engines are available and their paths.

2. Check Permissions

403 errors mean the token lacks permission. Check which policy is needed for the operation.

3. Verify Paths

404 errors usually mean a typo or missing mount. KV v2 paths use secret/data/ internally.

4. Check PKI Chain

Verify issuer exists, role is configured, and TTLs are within the issuer's max TTL.
